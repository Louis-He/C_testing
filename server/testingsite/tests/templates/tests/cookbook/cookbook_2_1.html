Data Type

Everything has a type. For example, apple is a fruit; your computer is an electronics; water is a type of liquid. This goes the same way when we are in the programming world. All the variables have their own types. So, let's meet some basic types we need to know. 

int a;     //This is a signed integer
double b;  //This is a double type floating point
char c;    //This is a signed character

Data is stored inside the computer memory. However, different kinds of data may require different size of space to store. You know, it's just like you cannot park your car in your refrigerator. In C and C++, develpers need to tell the computer explicitly what data type they are using so that they can be stored in a proper space.

On the other hand, different kinds of data may be used in different ways. Computer will not know what to do with the data you give to it unless you define the data type.

In general, the basic data types can be divided into two categories: the integer type and the floating type . Don't worry, we will go through all the details right away.

Bit and Byte

Before we start talking about data type, we need to learn a little bit about how numbers are stored in a computer.

Instead of using decimal numbers, mordern computers are using binary numbers to store data. This simply means that a digit in a binary number can be only 0 or 1. In fact, this is what we call one bit: a bit is essentailly a binary digit with two possible states: 0 and 1.

With a sequence of bits, computer will be able to represent numbers and perform arithmetic or logic operations. However, dealing with a bunch of bits are not convenient. Conventionally, we define that a group of bits with size of 8 as one byte. Therefore, the relation between bit and byte is:

1 Byte = 8 Bits

Integer Type

Integer type is used to store integer data. Even though some of them may have different use, for example, a char can be used to represent a character, but it is stored as an integer in the memory space.

The following table shows all the commonly used integer types, as well as the space each type takes and the range of integer it can store.

Type Memory Space Range
bool 1 bit 0 or 1
char 1 byte -128 to 127
unsigned char 1 byte 0 to 255
int 4 bytes -2,147,483,648 to 2,147,483,647
unsigned int 4 bytes 0 to 4,294,967,295
short 2 bytes -32,768 to 32,767
unsigned short 2 bytes 0 to 65,535
long 8 bytes -9,223,372,036,854,775,808 to 9,223,372,036,854,775,807
unsigned long 8 bytes 0 to 18,446,744,073,709,551,615

Note:

The size of each type may be different under different computer architecture, or with different compilier. The table above shows the memory space under x86_64 instruction set with Linux gcc-5.3.1 compiler. At this moment, you don't have to understand what those stuffs are, you just need to have a sense that the space of given type is machine dependent.

To get a precise memory size of a given type, you can use sizeof operator. For example: 

sizeof(int)

This expression will return the how many bytes an integer takes in your computer.

Floating Type

Integers are not suitable for all applications. Sometimes it is necessary to have variables that can store numbers with digits after the decimal point. These variables are called floating type.

The following table lists all the most commonly used floating types. The difference of these types is the precision of representing decimal digits. Similar to the integer type, more precise the type is, more memory space it will occupy. Note that all the floating types are signed numbers.

Type Memory Space Range (Absolute value) Precision
float 4 bytes 1.2e-38 to 3.4e+38 6 decimal place
double 8 bytes 2.3e-308 to 1.7e+308 15 decimal place
long double 16 bytes 3.4e-4932 to 1.1e+4932 19 decimal place
